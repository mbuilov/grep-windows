Instructions how to build native grep.exe with Microsoft Visual Studio from CYGWIN shell.

While building, it is possible to save build log and create a patch - for compiling grep.exe using only Microsoft Visual Studio tools, without need for CYGWIN.

This is how grep-3.3-build-VS17-x64.patch was created.

(grep-3.3-build-VS9-x86.patch was created analogously in Microsoft Visual Studio 2008 environment)


From CYGWIN shell:

1) get archive:
  wget https://ftp.gnu.org/gnu/grep/grep-3.3.tar.xz

2) unpack archive:
  tar xf grep-3.3.tar.xz

3) go to grep sources:
  cd grep-3.3

4) fix sources for windows:
  patch -Np1 -i grep-3.3-src.patch
  (patch location: https://raw.githubusercontent.com/mbuilov/grep-windows/master/grep-3.3-src.patch)

;5) to build with old non-c99 compiler (such as Microsoft Visual Studio 2008) apply one more patch:
;  patch -Np1 -i grep-3.3-src-non-c99.patch
;  (patch location: https://raw.githubusercontent.com/mbuilov/grep-windows/master/grep-3.3-src-non-c99.patch)

6) now start dos prompt:
  cmd.exe /c "start cmd.exe"

7) setup compiler:
  "D:\Program Files (x86)\Microsoft Visual Studio\2017\Community\VC\Auxiliary\Build\vcvarsall.bat" x86_amd64
  (if Microsoft Visual Studio 2017 is installed in "D:\Program Files (x86)")

;  --tip: for Visual Studio 2008:
;  "C:\Program Files (x86)\Microsoft Visual Studio 9.0\VC\vcvarsall.bat" x86

8) run bash from dos prompt (with environment prepared for compiling with Microsoft Visual Studio):
  bash

9) check Gnu Make is installed in CYGWIN and it is working:
  make --version

10) configure grep:
  CFLAGS= CC=cl CPP="cl -E" LD=lib ARFLAGS="/OUT:" ./configure --enable-threads=windows --disable-dependency-tracking --disable-silent-rules --disable-nls

11) configure script is not fully functional to create correct Makefiles - they are need to be fixed manually:
  sed -i 's/^SUBDIRS = .*/SUBDIRS = lib src/' ./Makefile
  sed -i '/^NEXT/s/=.*/=""/'                  ./Makefile ./lib/Makefile ./src/Makefile
  sed -i 's/^RECURSIVE_TARGETS = /&gen /'     ./Makefile
  sed -i 's/$(MAKE) $(AM_MAKEFLAGS) "$$target-am" || exit 1/:/' ./Makefile
  sed -i 's/-c -o /-c -nologo -Fo/'           ./lib/Makefile ./src/Makefile
  sed -i 's@^AR = .*@AR = lib /nologo@'       ./lib/Makefile
  sed -i '/libgreputils_a_AR/s/ lib/lib/'     ./lib/Makefile
  sed -i 's%^am_libgreputils_a_OBJECTS = %&getpagesize.$(OBJEXT) stat-w32.$(OBJEXT) %' ./lib/Makefile
  sed -i 's%^LINK = .*%LINK = link /nologo /DEFAULTLIB:LIBCPMT.lib /OUT:$@%'           ./src/Makefile

12) add gen target:
  echo 'gen:' >> ./Makefile
  echo 'gen:' >> ./src/Makefile
  echo 'gen: $(BUILT_SOURCES)' >> ./lib/Makefile

13) execute Makefile to generate grep headers:
  make gen

14) some references to system header files are missing in generated headers - they need to be fixed manually.

 a) set paths to locations of Visual Studio 2017 and WDK10 in grep-3.3-win-headers.patch (assume it was copied to the current directory):
  for example, if Visual Studio 2017 is installed in "D:\Program Files (x86)" and WDK10 installed in "C:\Program Files (x86)"

  sed -i '/Visual Studio/s@.:.*include@D:\\Program Files (x86)\\Microsoft Visual Studio\\2017\\Community\\VC\\Tools\\MSVC\\14.16.27023\\include@' ./grep-3.3-win-headers.patch
  sed -i '/Windows Kits/s@.:.*ucrt@C:\\Program Files (x86)\\Windows Kits\\10\\Include\\10.0.17763.0\\ucrt@' ./grep-3.3-win-headers.patch

  (patch location: https://raw.githubusercontent.com/mbuilov/grep-windows/master/grep-3.3-win-headers.patch)

;  --tip: for Visual Studio 2008:
;  sed -i '/Visual Studio/s@.:.*include@C:\\Program Files (x86)\\Microsoft Visual Studio 9.0\\VC\\include@' ./grep-3.3-win-headers.patch
;  sed -i '/Windows Kits/s@.:.*ucrt@C:\\Program Files (x86)\\Microsoft Visual Studio 9.0\\VC\\include@' ./grep-3.3-win-headers.patch

 b) now patch generated headers with updated patch-file:
  patch -Np1 -i grep-3.3-win-headers.patch

 c) header file lib/unistd.h is generated erroneously, fix it:
  patch -Np1 -i grep-3.3-unistd.patch
  (patch location: https://raw.githubusercontent.com/mbuilov/grep-windows/master/grep-3.3-unistd.patch)

15) do compile grep:
  make > make.bat

16) check build result:
  ./src/grep.exe --version

  (should print grep version, e.g.: (GNU grep) 3.3)

compilation should be ok, native (unoptimized) ./grep/grep.exe should be created.


Now it is possible to create a patch file - for compiling optimized grep.exe using only Microsoft Visual Studio tools, without need for CYGWIN.

1) create directory for unpatched grep:
  mkdir ../orig

2) unpack grep:
  tar xf ../grep-3.3.tar.xz -C ../orig/

3) diff current directory with original just unpacked grep-3.3.tar.xz in '../orig' directory
  diff -rql . ../orig/grep-3.3

4) remove unneeded built files in the current directory (object files, libs, etc...)

  rm -r \
    ./Makefile \
    ./stamp-h1 \
    ./grep-3.3-win-headers.patch \
    ./src/*.obj \
    ./lib/*.obj \
    ./lib/glthread/*.obj \
    ./lib/glthread/.deps \
    ./lib/glthread/.dirstamp \
    ./lib/unistr/*.obj \
    ./lib/unistr/.deps \
    ./lib/unistr/.dirstamp \
    ./lib/uniwidth/*.obj \
    ./lib/uniwidth/.deps \
    ./lib/uniwidth/.dirstamp \
    ./lib/libgreputils.a \
    ./src/grep.exe \
    ./src/egrep \
    ./src/fgrep \
    ./src/Makefile \
    ./lib/Makefile \
    ./doc/Makefile \
    ./tests/Makefile \
    ./gnulib-tests/Makefile \
    ./po/Makefile \
    ./po/Makefile.in \
    ./po/POTFILES \
    ./config.log \
    ./config.status

5) after this, diff should show something like:

  diff -rql . ../orig/grep-3.3
  Only in .: config.h
  Only in ./lib: alloca.h
  Only in ./lib: colorize.c
  Only in ./lib: configmake.h
  Only in ./lib: ctype.h
  Only in ./lib: dirent.h
  Only in ./lib: errno.h
  Only in ./lib: fcntl.h
  Only in ./lib: fnmatch.h
  Files ./lib/fts.c and ../orig/grep-3.3/lib/fts.c differ
  Only in ./lib: getopt.h
  Only in ./lib: getopt-cdefs.h
  Only in ./lib: inttypes.h
  Only in ./lib: langinfo.h
  Only in ./lib: limits.h
  Only in ./lib: locale.h
  Only in ./lib: signal.h
  Only in ./lib: stdalign.h
  Only in ./lib: stddef.h
  Only in ./lib: stdint.h
  Only in ./lib: stdio.h
  Only in ./lib: stdlib.h
  Only in ./lib: string.h
  Only in ./lib: sys
  Only in ./lib: time.h
  Only in ./lib: unistd.h
  Only in ./lib: unistr.h
  Only in ./lib: unitypes.h
  Only in ./lib: uniwidth.h
  Only in ./lib: wchar.h
  Only in ./lib: wctype.h
  Only in .: make.bat
  Files ./src/grep.c and ../orig/grep-3.3/src/grep.c differ

;If patch grep-3.3-src-non-c99.patch was applied, diff will show additional differences:
;

6) edit make.bat

  cp make.bat make.bat.old
  sed -i -n '/^cl /p;/^lib /p;/^link /p' make.bat
  sed -i '/.nologo /s@/@\\@g' make.bat
  sed -i '/-DHAVE_CONFIG_H -I. -I..       -c/s@-Fo@&lib\\@' make.bat
  sed -i '/-DHAVE_CONFIG_H -I. -I..       -c/s@.cygpath -w .@lib\\@' make.bat
  sed -i '/-DHAVE_CONFIG_H -I. -I..  -I...lib -I...lib      -c/s@-Fo@&src\\@' make.bat
  sed -i '/-DHAVE_CONFIG_H -I. -I..  -I...lib -I...lib      -c/s@.cygpath -w .@src\\@' make.bat
  sed -i 's@-DHAVE_CONFIG_H -I. -I..  -I..\\lib -I..\\lib .* -c@/I. /I.\\lib /I.\\src -c@' make.bat
  sed -i 's@-DHAVE_CONFIG_H -I. -I.. .* -c@/I. /I.\\lib -c@' make.bat
  sed -i 's@-c -nologo -Fo@/Ox /GF /Gy /GS- /GL /EHsc /DHAVE_CONFIG_H /c /nologo /Fo@' make.bat
  sed -i 's@.`@@' make.bat
  sed -i '/^lib/s@ .nologo .OUT:@@;/^lib/s@  *$@@g;/^lib/s@ @ .\\lib\\@g;s@^lib@lib /LTCG /nologo /OUT:@' make.bat
  sed -i '/^link/s@..\\lib\\libgreputils.a@@g' make.bat
  sed -i '/^link/s@ .nologo .DEFAULTLIB:LIBCPMT.lib .OUT:@@;/^link/s@  *$@@g;/^link/s@ @ .\\src\\@g' make.bat
  sed -i 's@^link@link /LTCG /nologo /DEFAULTLIB:LIBCPMT.lib .\\libgreputils.a /OUT:@' make.bat
  rm make.bat.old

7) finally, create a patch:
  diff -Naur ../orig/grep-3.3 . > ../grep-3.3-build-VS17-x64.patch

; or, for Visual Studio 2008:
;  diff -Naur ../orig/grep-3.3 . > ../grep-3.3-build-VS9-x86.patch
